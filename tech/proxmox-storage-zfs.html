Proxmox: Storage Choices, ZFS, and Replication

Storage is where “hypervisor” turns into “architecture.” Many Proxmox clusters use local ZFS plus replication for a simple, reliable baseline.

Why ZFS is common

  • Fast snapshots
  • Checksums to catch silent corruption
  • Compression for better effective capacity
  • Datasets for tuning

Replication is not shared storage

Replication is asynchronous: if a node fails, you may lose the last few minutes depending on schedule. That can be totally acceptable, but you should decide intentionally.

Practical tips

  • Prefer mirrored vdevs for rebuild predictability
  • Don’t guess: measure ZFS ARC + guest memory usage
  • Test restores, not just backups
